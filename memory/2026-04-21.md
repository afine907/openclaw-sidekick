## Nano Code CLI Major Refactor - TypeScript Migration

### Context
- **Project**: afine907/nano-code (our maintained repo)
- **Goal**: Align CLI experience with Claude Code by migrating from Python/rich to TypeScript/ink

### Architecture Decision
Migrated CLI layer to TypeScript while keeping Python core unchanged:
- **CLI**: TypeScript + ink (React for CLI)
- **Communication**: JSON-RPC 2.0 over stdio
- **Core**: Python (LangGraph Agent + Tools + LLM) - unchanged

### Completed Work

#### Phase 1: Infrastructure (30 min)
- Created monorepo structure with pnpm workspace
- Python JSON-RPC Server: `src/nano_code/server/`
  - `protocol.py`: JSON-RPC types (Request, Response, Notification, StreamEvent)
  - `rpc.py`: Server implementation with methods: agent.stream, tools.list, tool.execute, config.get
- TypeScript JSON-RPC Client: `packages/cli/src/client/`
  - `types.ts`: Type definitions
  - `rpc.ts`: Client with EventEmitter for stream events

#### Phase 2: CLI Components (20 min)
- `ChatView.tsx`: Message list with welcome screen
- `MessageItem.tsx`: Individual message display
- `InputBox.tsx`: Multi-line input with ink-text-input
- `StatusBar.tsx`: Model and status display
- `Markdown.tsx`: Markdown rendering with code blocks
- `Animation.tsx`: ToolExecution and ThinkingIndicator
- `useAgent.ts`: React hook for agent state management

#### Phase 3: Integration (10 min)
- Main app in `packages/cli/src/index.tsx`
- Integrated all components with useAgent hook

### Key Technical Decisions
1. **JSON-RPC over stdio**: Simple, reliable IPC between TypeScript and Python
2. **EventEmitter pattern**: Clean stream event handling
3. **Monorepo**: pnpm workspace for clean separation
4. **Type safety**: Full TypeScript types for all interfaces

### File Structure Created
```

├── tsconfig.base.json

│   ├── src/

│   │   ├── client/ (RPC)
│   │   ├── components/ (UI)
│   │   └── hooks/ (useAgent)
│   └── package.json
└── src/nano_code/server/ (JSON-RPC)
```

### PR
- PR #13: https://github.com/afine907/nano-code/pull/13
